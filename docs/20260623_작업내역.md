상단 메뉴 `대산 블로그`에 실행 잠금 해제 메뉴를 추가했다.


```text

PUBLISH_RUNNING
```

## 6. 이미지 오류 로그 개선

OpenAI 이미지 생성 실패 원인을 확인하기 위해 로그를 강화했다.

- `generateImageWithOpenAI_`에서 `error.message` 전체 기록
- `error.stack` 전체 기록
- OpenAI `responseCode` 기록
- OpenAI `responseText` 전체 기록
- GitHub 업로드 응답 body 전체 기록
- `Logger.log(error)` 추가
- `console.error(error.stack)` 추가

G열 상태 칸에는 단순 `오류` 대신 다음 형식으로 상세 원인을 기록하도록 변경했다.

```text
OpenAI 오류: ...
GitHub 오류: ...
이미지 오류: ...
```

## 7. OpenAI API Key 설정

코드는 Script Properties에서 다음 키를 읽는다.

```text
OPENAI_API_KEY
```

확인된 코드 기준 키 이름:

```js
OPENAI_API_KEY: 'OPENAI_API_KEY'
PropertiesService.getScriptProperties().getProperty(PROP_KEYS.OPENAI_API_KEY)
```

실제 키 값은 문서와 로그에 기록하지 않는다.

현재 이미지 생성 모델명:

```text
gpt-image-2
```

모델 변경은 사용자 승인 전에는 하지 않는다.

## 8. GitHub 업로드 구조 개선

이미지 업로드 대상 저장소 하드코딩을 제거하고 Script Properties 기반 설정으로 분리했다.

제거한 하드코딩 대상:

```text

```

새 Script Properties 키:

```text

GITHUB_OWNER
GITHUB_REPO
GITHUB_BRANCH
GITHUB_IMAGE_DIR
```

기본값:

```text
GITHUB_OWNER = Jangho0611
GITHUB_REPO = blogger_jh
GITHUB_BRANCH = main
GITHUB_IMAGE_DIR = images
```

현재 코드 기준 이미지 업로드는 사본 저장소 `Jangho0611/blogger_jh`를 기본 대상으로 사용한다.

## 10. 현재 운영 원칙

- 원본 GitHub 저장소에는 push하지 않는다.
- 작업 결과는 `origin/main`에만 push한다.
- `upstream/main`은 원본 추적 및 비교용으로만 사용한다.
- 원본 Google Sheet `Main_ver20_blogger`는 읽기 전용으로 간주한다.
- 모든 개발/테스트는 `Main_ver20_blogger의 사본`에서 수행한다.
- Apps Script 배포 대상은 `.clasp.json`의 새 `scriptId`만 사용한다.
- `clasp pull`은 사용하지 않는다.
- 강제 push 전에는 `.clasp.json`, `CONTROL_SHEET_ID`, 서버 반영 상태를 확인한다.
- API 키, 토큰, OAuth credential 값은 출력하지 않는다.
